FAQ

What is the Document Object Model (DOM)?

DOM stands for Document Object Model and is a programming interface that allows us to create, change or remove elements from the document. We can also add events to these elements to make our page more dynamic.

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a web page in a tree-like format, allowing programs and scripts to dynamically access and manipulate its content, structure, and style.

How do you select an element from the DOM?

Selecting elements from the DOM is a common task in web development. JavaScript provides various methods to access and manipulate elements based on their tag name, class, ID, attributes, and more. Below are the most commonly used methods:

  • 1. Selecting by ID:
    Use the document.getElementById() method to select an element by its unique id attribute.
  • 2. Selecting by Class Name:
    Use document.getElementsByClassName() to select elements by their class attribute. This method returns a live HTMLCollection (similar to an array).
  • 3. Selecting by Tag Name:
    Use document.getElementsByTagName() to select elements by their tag name (e.g., div, p, ul). This method also returns a live HTMLCollection.
  • 4. Selecting by CSS Selector:
    Use document.querySelector() to select the first element that matches a CSS selector (e.g., #id, .class, div, etc.).
  • 5. Selecting Multiple Elements by CSS Selector:
    Use document.querySelectorAll() to select all elements that match a CSS selector. This returns a static NodeList (which can be iterated).
  • 6. Selecting by Attribute:
    You can use querySelector or querySelectorAll to select elements by their attributes. For example, selecting elements with a specific attribute (like data-* attributes).
What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is attached to a parent element to handle events from its child elements. Instead of adding individual listeners to each child, the event bubbles up from the target element to the parent, where it’s handled.

This is useful because it improves performance by reducing the number of event listeners, especially in dynamic lists or elements that are frequently added or removed. It also simplifies code, ensuring all future child elements are automatically handled without needing additional listeners. For example, using addEventListener on a parent element.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes in the DOM, use getAttribute() to retrieve, setAttribute() to modify, and removeAttribute() to delete attributes. For common attributes like id and className, modify them directly (e.g., element.id = 'newId').

To change an element’s styles, use the style property for inline styles (e.g., element.style.color = 'red') or manipulate classes using classList.add(), remove(), or toggle().

Data attributes (e.g., data-*) can be accessed and modified via dataset (e.g., element.dataset.user = '123').
